游戏指南

• 使用 ↑ ↓ ← → 方向键移动方块
• 当两个相同数字的方块碰撞时，它们会合并成它们的和
• 每次移动后，会在空白位置随机生成一个2或4
• 当网格被填满且无法移动时，游戏结束
• 点击"撤销"按钮可以回退到上一步操作

创建游戏 CDN 链接的步骤

1创建 GitHub 仓库

在 GitHub 上创建一个新的仓库来存放你的游戏文件：

  • 访问 github.com/new
  • 输入仓库名称（例如：my-2048-game）
  • 选择公开（Public）
  • 勾选 "Initialize this repository with a README"
  • 点击 "Create repository"

2上传游戏文件

将你的游戏 HTML 文件上传到仓库：

  • 在仓库页面，点击 "Add file" → "Upload files"
  • 将你的 HTML 文件拖到上传区域
  • 添加提交信息（例如：Add game file）
  • 点击 "Commit changes"

示例仓库结构

my-2048-game/
├── index.html   # 你的游戏主文件
└── README.md    # 仓库说明文件

3获取 jsDelivr CDN 链接

jsDelivr CDN 链接格式如下：

https://cdn.jsdelivr.net/gh/你的GitHub用户名/仓库名/文件路径

例如：

如果你的用户名是 game-dev，仓库名是 2048-game，文件是 index.html，则 CDN 链接为：

https://cdn.jsdelivr.net/gh/game-dev/2048-game@main/index.html

注意：

  • 将 main 替换为你的仓库分支名称（默认为 main）
  • 如果你的文件在子目录中，需要包含完整路径

4分享你的游戏

现在你可以分享这个链接给任何人：

  • 链接会通过 jsDelivr 的全球 CDN 加速
  • 用户无需下载即可直接在浏览器中玩游戏
  • 更新游戏只需在 GitHub 上更新文件

这就是你提供的俄罗斯方块 CDN 链接的工作原理：

https://cdn.jsdelivr.net/gh/Uharasakura/-@main/Tetris.html

5高级技巧

优化你的 CDN 链接：

  • 版本控制：使用版本号而不是分支名，确保链接稳定
  • 子目录：将游戏放在子目录中保持仓库整洁
  • 压缩资源：压缩 CSS/JS 文件减少加载时间

版本化链接示例：

https://cdn.jsdelivr.net/gh/user/repo@1.0.0/file.js
